## 🔄 文档更新流程

### 何时更新文档

1. **完成新功能** → 更新 `PROJECT_STATUS.md` 和 `DEVELOPMENT_LOG.md`
2. **修复重要 Bug** → 记录到 `DEVELOPMENT_LOG.md`
3. **架构变更** → 更新 `docs/architecture/infrastructure.md`
4. **API 变更** → 更新 `API_USAGE_GUIDE.md` 和 `docs/frontend-backend-integration.md`
5. **部署流程变更** → 更新 `docs/deployment/deployment-guide.md`
6. **会话结束** → 更新 `SESSION_HANDOFF.md`

## 📝 文档编写规范

### Markdown 格式规范
- 使用 ATX 风格标题（`#` 而不是下划线）
- 代码块必须指定语言
- 使用表格组织结构化数据
- 使用 emoji 增强可读性（适度使用）

### 文档结构规范
```markdown
# 文档标题

**元信息**: 版本、日期、作者等

---

## 概述
简短描述文档目的

## 主要内容
...

## 参考资料
相关文档链接

---

**维护者**: 姓名
**最后更新**: 日期
```

### 命名规范
- 使用大写字母和下划线：`PROJECT_STATUS.md`
- 日期前缀格式：`YYYY-MM-DD-topic.md`
- 使用描述性名称，避免缩写
